# Security Policy

| Version | Supported          |

## Reporting a Vulnerability

We take the security of ENZIU seriously. If you believe you have found a security vulnerability, please report it to us as described below.

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to **security@enziu.com** with the following information:

1. Description of the vulnerability
2. Steps to reproduce the issue
3. Potential impact
4. Any suggested fixes (if applicable)

You should receive a response within 48 hours acknowledging your report. After the initial reply, we will keep you informed of the progress toward a fix and announcement.

## Security Measures

ENZIU implements the following security measures:

### Data Privacy
- **Zero disk write**: All PDFs are processed entirely in memory using `io.BytesIO`
- **No PII storage**: Voucher system uses code + bcrypt passphrase only, no email required
- **Session cleanup**: All session data is permanently deleted when user closes the tab
- **No data retention**: User documents are never stored or logged

### Authentication & Authorization
- **HMAC validation**: Fast rejection of fake voucher codes before database lookup
- **Bcrypt passphrase hashing**: All passphrases are hashed with bcrypt (cost factor 12)
- **Atomic operations**: Credit decrement uses Redis atomic operations to prevent double-spending

### Infrastructure Security
- **HTTPS only**: All production traffic enforced via HSTS headers
- **Security headers**: 
  - `Strict-Transport-Security`: 2 year max-age with preload
  - `X-Frame-Options`: SAMEORIGIN
  - `X-Content-Type-Options`: nosniff
  - `Referrer-Policy`: strict-origin-when-cross-origin
  - `Cross-Origin-Embedder-Policy`: require-corp
  - `Cross-Origin-Opener-Policy`: same-origin
- **CORS protection**: Strict origin validation for API endpoints
- **Rate limiting**: API rate limiting to prevent abuse

### Dependency Management
- **Regular updates**: All dependencies are updated monthly
- **Vulnerability scanning**: Automated security scanning via GitHub Dependabot
- **Lock files**: Both `package-lock.json` and `poetry.lock` ensure reproducible builds

### Code Quality
- **Type safety**: Full TypeScript strict mode and Python type hints
- **Static analysis**: Ruff for Python, ESLint for TypeScript
- **Code review**: All changes require peer review before merging

## Known Limitations

1. **Not legal advice**: ENZIU provides analysis, not legal advice. All outputs include disclaimers.
2. **AI limitations**: Analysis is generated by AI and may contain inaccuracies.
3. **Document scope**: Only insurance policies in PDF format are supported.

## Security Updates

Security updates will be released as patch versions (e.g., 1.0.1). Critical security fixes may be released as needed.

Subscribe to our security mailing list at **security@enziu.com** to receive notifications about security updates.

## Recognition

We appreciate responsible disclosure and will acknowledge researchers who help improve our security (with permission).
